# Contributing Guide

Miku's Origin 公式サイトへのコントリビューションありがとうございます。

本プロジェクトはサークルメンバーによる共同管理を想定しています。

---

# 開発環境セットアップ

```bash
npm install
npm run dev
```

# コーディングルール

### Format

Prettierで自動整形します

```bash
npm run format
```

### Lint

```bash
npm run lint
```

エラーが出ない状態かつサイトが想定通り表示されることを確認して、PRを出してください。

# ブランチルール

- `main`ブランチに対して直接プッシュしないこと
- `dev`ブランチから機能ごとにブランチを切ること

```bash
例
main-----------------------------
    \-dev------------------------
            \-feat/footer---/
```

# コミットメッセージルール

本リポジトリでは以下のメッセージプレフィックスを採用しています。<br />
参照：[Conventional Commits](https://www.conventionalcommits.org/ja/v1.0.0/)
| prefix | 内容 | 例 |
| -------- | ----------------------- | --------------------------- |
| feat | 新機能追加 | feat: メンバー一覧ページを追加 |
| fix | バグ修正 | fix: モバイル表示で崩れるレイアウトを修正 |
| docs | ドキュメント変更 | docs: READMEにセットアップ手順を追記 |
| style | フォーマット・見た目のみの変更（動作影響なし） | style: prettierでコード整形 |
| refactor | リファクタリング（仕様変更なし） | refactor: コンポーネント構成を整理 |
| perf | パフォーマンス改善 | perf: 画像最適化で読み込み速度改善 |
| test | テスト追加・修正 | test: ヘッダーコンポーネントのテスト追加 |
| build | ビルド設定・依存関係変更 | build: tailwindの設定更新 |
| ci | CI/CD設定変更 | ci: GitHub Actionsのワークフロー修正 |
| chore | 雑務（本番影響なし） | chore: 不要ファイル削除 |

### 推奨フォーマット

```bash
prefix: 変更内容を簡潔に書く
```

# PR前チェック事項

- ビルドがエラーなく通る
- lintエラーなし
- 型エラーなし
- ページが想定通り表示される
- 意図していないページに意図していない変更がない

# デザイン変更について

大きなデザイン変更やUI変更は事前に相談してください
